## To initialise and push

```powershell
# From D:\PTCG
git init
git branch -m main

# Verify .gitignore is doing its job before staging
git status --short | head -40
# (you should NOT see venv/, node_modules/, .next/, .pdf, .pt, or .egg-info entries)

git add .
git commit -m "Initial commit: production-ready Pokémon TCG AI framework"

# Create the empty repo on github.com first (no README, no .gitignore — we have our own)
# Then:
git remote add origin git@github.com:YOUR-USERNAME/pokemon-tcg-ai.git
# or HTTPS:
# git remote add origin https://github.com/YOUR-USERNAME/pokemon-tcg-ai.git

git push -u origin main
```

## After the first push — recommended GitHub settings

1. **Settings → General → Default branch:** `main`.
2. **Settings → Branches → Branch protection rule on `main`:**
   - Require status checks to pass before merging (select the CI workflow: `Tests / lint / coverage`, `Frontend / typecheck / build`, `Docker image`).
   - Require a pull request before merging.
3. **Settings → Actions → General:**
   - Allow GitHub Actions to read/write the repo (needed for upload-artifact).
4. **Settings → Code security:**
   - Enable Dependabot alerts.
   - Enable secret scanning (optional but recommended).
5. **About** sidebar on the repo page:
   - Description: *"Pokémon TCG AI — full Scarlet & Violet rules engine + PUCT MCTS + Next.js frontend"*
   - Topics: `pokemon-tcg`, `mcts`, `alphazero`, `reinforcement-learning`, `fastapi`, `nextjs`, `kaggle`.

## Sanity test after pushing

```powershell
# Fresh clone in a different folder
git clone git@github.com:YOUR-USERNAME/pokemon-tcg-ai.git /tmp/fresh-clone
cd /tmp/fresh-clone
python -m pip install -e ".[dev,server]"
python -m pytest -q
python -m src.cli --json benchmark
```

All three should succeed (the test suite uses CSV-availability skips for two tests if `EN_Card_Data.csv` is absent — copy it manually or add it as a Git LFS object if you want CI to find it).

Note: `EN_Card_Data.csv` (359 KB) **is** committed; it's the runtime card database the engine needs. Only the 137 MB PDF is excluded.

## If you want to ship a trained checkpoint

Use **GitHub Releases** (or a separate Kaggle Dataset / Hugging Face repo) — do not commit `*.pt` files. Workflow:

```powershell
# After producing ckpt_round_8.pt on Kaggle (see submission/kaggle_notebook.ipynb)
gh release create v1.0.0 \
  --title "Pokémon TCG AI v1.0.0" \
  --notes-file PRODUCTION_HARDENING_REPORT.md \
  ckpt_round_8.pt
```

Document the download URL in `README.md` and `MODEL_SELECTION.md`.
